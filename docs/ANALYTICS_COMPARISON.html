מול דרישות PQ (Pre-Qualification) של מערכות ניהול שירותים חכמים

קיים ומלא קיים חלקית לא קיים
1. מדדי ביצוע מרכזיים (KPIs) — לוח בקרה
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
1.1 סה"כ קריאות (כולל / פתוחות / בטיפול / טופלו) ✔ נדרש ✔ קיים כרטיסי סיכום בדשבורד: openIncidents, inProgress, resolved, total. סינון לפי בניין.
1.2 זמן תגובה ממוצע (Resolution Time) ✔ נדרש ✔ קיים avgResolutionMinutes — ממוצע 30 יום אחרונים. מוצג בדשבורד + דו"ח יומי.
1.3 עמידה ב-SLA (אחוז קריאות שטופלו בזמן) ✔ נדרש ✔ קיים slaPercent עם יעד מתכוונן (10/15/20/30 דק'). גרף Gauge + p50/p90. גם בדו"ח יומי (15 דק').
1.4 מספר מבקרים / תנועת קהל (People Counting) ✔ נדרש ✘ לא קיים פער: המערכת לא מונה כניסות/יציאות. נדרש חיישן תנועה / PIR ייעודי.
1.5 שעות שיא שימוש (Peak Hours) ✔ נדרש ◐ חלקי גרף שעתי (Hourly) מראה התפלגות קריאות לפי שעה — לא תנועת קהל ישירה.
1.6 התפלגות לפי ימי שבוע (Day-of-Week) ◐ רצוי ✔ קיים גרף עמודות לפי יום בשבוע עם מספר קריאות.
1.7 מכשירים מקוונים / לא מקוונים ✔ נדרש ✔ קיים onlineDevices, offlineDevices + lastHeartbeat + רשימה מפורטת בדשבורד.
1.8 עובדי ניקיון פעילים במשמרת ✔ נדרש ✔ קיים activeCleaners — על סמך CleanerArrival. כרטיס בדשבורד + דף עובדים.
2. ביצועי עובדי ניקיון
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
2.1 דירוג עובדים לפי מספר טיפולים ✔ נדרש ✔ קיים Cleaner Performance — רשימת עובדים ממוינת לפי totalResolved עם גרף פס.
2.2 זמן תגובה ממוצע לעובד ✔ נדרש ✔ קיים avgResolutionMinutes לכל עובד בנפרד.
2.3 זיהוי עובדים סרק (Idle Cleaners) ◐ רצוי ✔ קיים idleCleaners — 30+ דקות במשמרת ללא טיפול אחד. מופיע בדו"ח יומי + Mismatch בדף עובדים.
2.4 צ'ק-אין / צ'ק-אאוט עובדים ✔ נדרש ✔ קיים CleanerArrival — שעת הגעה, משך משמרת, סטטוס נוכחי בזמן אמת.
2.5 הוכחת ביצוע עם תמונות (Photo Proof) ✔ נדרש ✘ לא קיים פער: אין צילום לפני/אחרי ניקיון. נדרש פיתוח העלאת תמונות.
2.6 רשימת משימות דיגיטלית (Digital Checklist) ✔ נדרש ◐ חלקי עובד רואה קריאות פתוחות לטיפול ויכול לסמן "טופל" — אין רשימת ביקורת מפורטת (צ'קליסט).
3. ניהול קריאות והיסטוריה
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
3.1 היסטוריית פעולות לכל קריאה (Audit Trail) ✔ נדרש ✔ קיים IncidentAction — דווח → אושר → הוקצה → טופל/הוסלם. כולל שם עובד, הערות, שעה.
3.2 הסלמה אוטומטית (Escalation) ✔ נדרש ✔ קיים escalationRound + Push Notifications. התראות ברמת עובד → מנהל משמרת → מנהל.
3.3 קריאות דחופות (Urgent Alerts) ✔ נדרש ✔ קיים חישוב minutesOpen + סף מוגדר. מוצג בדשבורד ובעמוד בקשות עם אנימציית pulse.
3.4 תדירות סוגי בעיות (Issue Frequency) ✔ נדרש ✔ קיים גרף עמודות אופקי לפי סוג בעיה — מספר קריאות + זמן טיפול ממוצע לכל סוג.
3.5 דפוסים חוזרים ונקודות חמות (Patterns & Hotspots) ✔ נדרש ✔ קיים Top 5 סוגי בעיות + Top 5 שירותים עם הכי הרבה קריאות. דגל "מעל הממוצע".
3.6 משוב חיובי (Positive Feedback) ◐ רצוי ✔ קיים כפתור "עבודה טובה" בקיוסק. משוב מופיע בנפרד בדף בקשות + נכלל בנתונים.
4. חיישנים וניטור סביבתי (IoT)
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
4.1 ניטור ריחות / איכות אוויר (אמוניה, H₂S) ✔ נדרש ✘ לא קיים פער: אין חיישני ריח/אוויר. נדרש חומרה + אינטגרציה.
4.2 ניטור מפלס מתכלים (סבון, נייר, מגבות) ✔ נדרש ✘ לא קיים פער: אין חיישני מפלס. דיווח מתבצע ידנית דרך כפתורי הקיוסק.
4.3 גילוי נזילות מים (Leak Detection) ✔ נדרש ✘ לא קיים פער: אין חיישני מים. נדרש חומרה.
4.4 טמפרטורה ולחות ◐ רצוי ✘ לא קיים פער: אין חיישני סביבה.
4.5 ספירת מבקרים (People Counter) ✔ נדרש ✘ לא קיים פער: אין חיישני ספירה / PIR. קריאות מייצגות דיווחי בעיות בלבד.
5. דיווחים וייצוא נתונים
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
5.1 דו"ח יומי אוטומטי במייל ✔ נדרש ✔ קיים HTML email אוטומטי: סיכום יום, SLA, Top issues, Top hotspots, ביצועי עובדים, עובדים סרק.
5.2 ייצוא ל-CSV / Excel ✔ נדרש ✘ לא קיים פער: אין כפתור הורדה/ייצוא לקבצי נתונים.
5.3 ייצוא ל-PDF ✔ נדרש ✘ לא קיים פער: אין הפקת דו"חות PDF מתוך המערכת.
5.4 דו"ח שבועי / חודשי מתוזמן ◐ רצוי ✘ לא קיים קיים רק דו"ח יומי. אין אגרגציה שבועית/חודשית מתוזמנת.
5.5 בחירת טווח תאריכים מותאם אישית ✔ נדרש ✔ קיים מסך אנליטיקס: 1/2/7/30/60 ימים + טווח from/to מותאם.
6. זמן אמת ומנשק קיוסק
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
6.1 עדכון בזמן אמת (WebSocket) ✔ נדרש ✔ קיים Socket.io — דשבורד, בקשות, קיוסק, ואפליקציית עובד מתעדכנים בזמן אמת.
6.2 סטטיסטיקות ציבוריות בקיוסק ◐ רצוי ✔ קיים קריאות השבוע, קריאות היום, זמן תגובה ממוצע — מאוחד לכל הבניין.
6.3 Push Notifications (Web Push / VAPID) ✔ נדרש ✔ קיים PWA עם Service Worker, Web Push Notifications דרך VAPID.
6.4 עבודה אופליין (Offline Mode) ◐ רצוי ✔ קיים קיוסק תומך בתור מקומי + סנכרון אוטומטי בחזרה לרשת.
6.5 ציון שביעות רצון (Guest Survey / Rating) ✔ נדרש ◐ חלקי כפתור "משוב חיובי" (😊) — אין סקאלה 1-5 או סקר מפורט.
7. יכולות מתקדמות
# מדד / סטטיסטיקה דרישת PQ ToiletCleanPro פירוט מימוש / פער
7.1 תחזוקה מנבאת (Predictive Maintenance) ✔ נדרש ✘ לא קיים פער: אין מודל חיזוי. ניתוח הוא תיאורי בלבד (ממוצעים, מגמות).
7.2 ניקיון מבוסס תפוסה (Occupancy-Based Cleaning) ✔ נדרש ✘ לא קיים פער: ללא חיישני תנועה — אין טריגר ניקיון לפי תפוסה.
7.3 חישוב חיסכון בעלויות (Cost Savings) ◐ רצוי ✘ לא קיים פער: אין חישוב עלויות כספיות או ROI.
7.4 Multi-tenant — ניהול מספר ארגונים ✔ נדרש ✔ קיים Organization → Building → Floor → Restroom. הרשאות לפי תפקיד.
7.5 תמיכה בריבוי שפות (i18n) ◐ רצוי ✔ קיים עברית + אנגלית. הגדרה ברמת ארגון לקיוסק ולממשק מנהל.
7.6 אינטגרציה עם CMMS (מערכת תחזוקה) ✔ נדרש ✘ לא קיים פער: אין ממשק API חיצוני או אינטגרציה עם מערכות תחזוקה.

פערים עיקריים לסגירה (לפי עדיפות)

  • חיישנים (IoT): ספירת מבקרים, ריחות, מפלס מתכלים, נזילות — דורש חומרה + אינטגרציה. זהו הפער הגדול ביותר.
  • ייצוא נתונים: CSV/Excel + PDF — פיתוח תוכנה בלבד, ניתן למימוש מהיר.
  • תחזוקה מנבאת: דורש מודל ML על נתונים היסטוריים — פיתוח ארוך טווח.
  • צ'קליסט + הוכחת ביצוע (תמונות): פיתוח תוכנה + אחסון תמונות.
  • סקר שביעות רצון מלא: הרחבת כפתור ה-Feedback לסולם 1-5 — פיתוח קצר.
  • אינטגרציית CMMS: ממשק API פתוח — בינוני.

יתרונות ToiletCleanPro (מעבר לדרישות ה-PQ)

  • PWA + Offline Mode: עבודה מלאה ללא רשת עם סנכרון אוטומטי — לא נפוץ במתחרים.
  • WebSocket Real-time: כל הממשקים מתעדכנים בזמן אמת ללא רענון.
  • קיוסקים מותאמים: מנוע תבניות (3 עיצובים), כפתורים מותאמים לפי ארגון.
  • משוב חיובי: ייחודי — מאפשר למבקרים לדווח גם על שביעות רצון.
  • דו"ח יומי אוטומטי: עם עובדים סרק (Idle Cleaners) — יכולת ייחודית.
  • אסקלציה אוטומטית: מערכת הסלמה רב-שלבית עם Push Notifications.
